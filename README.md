# ZANSIDE 스카우팅 보드 — OWCS ASIA: KOREA Stage 2

ZANSIDE 코칭스태프 **내부용** 전략·전술 대시보드. OWCS 경기 데이터를 구글시트에서 읽어 경기 전 준비에 쓴다. 외부 공개용이 아니다.

## ⚠️ 데이터 전제와 한계 (반드시 읽을 것 — 명세 15장)

- **세트 단위 5인 로스터가 거의 없다.** 현재 시트에는 세트별 5인 로스터(선픽)가 사실상 **1개 매치(6/5 O2 Blast전)에만** 있다. 따라서 아래 기능은 출시 시점에 대부분 **"데이터 부족"** 상태로 동작하며, 이것은 정상이다. 데이터가 쌓이면 자동으로 채워진다.
  - **승률 추정** (모의 로스터)
  - **선수** (선수별 분석 + 선수 비교)
- **표본이 작다.** 팀당 표본이 6경기 안팎이라 신뢰도가 낮다. 모든 승률·성적 옆에 **표본 수**를 함께 노출하며, 표본이 작을 때 단정적 표현·단일 확정 수치를 쓰지 않는다.
- **승률 추정은 모델이 아니다.** 학습된 예측 모델이 아니라 **투명한 가중 합산 휴리스틱**이다. 가중치는 코드 상수(`lib/constants.ts`의 `EST_WEIGHTS`)로 노출되며 화면 "요인 분해"에도 표시된다.

### 📌 데이터 확충 권장 (코치에게 요청)

선수 분석·승률 추정·영웅 메타의 **픽 화면**이 의미를 가지려면 **모든 세트의 양 팀 5인 로스터(선픽 컬럼)** 가 시트에 입력되어야 한다. 코치에게 **매 세트 양 팀 5인(역할별 선수명·영웅)을 `상수팀 첫픽 / 하수팀 첫픽` 컬럼에 입력**해 달라고 요청할 것. 컬럼 위치는 헤더 2행의 `딜러1 선수명`을 기준으로 자동 인식한다(상수팀·하수팀 블록 2곳). 픽 화면은 채워진 만큼 **자동으로** 채워진다.

- 기본 분석 범위는 **오프닝 선픽**이다. 세트 내 교체 영웅까지 보려면 별도 입력 칸(세트별 전체 등장 영웅)이 필요하다 — 옵션(§16.4·19.3).
- 밴 데이터는 모든 세트에 채워져 있어 **밴 관련 화면은 전부 지금 동작**한다.

## 데이터 소스

- 단일 원본: 구글시트 (코드에 데이터 하드코딩 없음)
- 사용 시트 3개: `OWCS 전체 공식경기`, `OWCS 대진표`, `OWCS 순위표`
- **`스크림 시트`는 읽지도 노출하지도 않는다** (민감 데이터)
- 읽기 방식: 메인 시트는 `export?format=csv`(푸시 거리값 보존), 실패 시 `gviz` CSV 폴백. 대진표·순위표는 `gviz` CSV.
- 시트 공유 설정이 **"링크가 있는 모든 사용자(보기)"** 여야 서버에서 fetch 된다.

## 기술 스택

- **Next.js 14 App Router** + **ISR**(`revalidate = 600`초). 데이터는 서버에서 fetch·파싱·파생 후 클라이언트로 직렬화 전달.
- 외부 런타임 의존성 없음(차트는 경량 SVG/막대 직접 구현, CSV 파서도 자체 구현).
- Vercel 배포 (`framework: nextjs`).

## 영웅 메타 모듈 (PART 2)

상위 메뉴 **"영웅 메타"** (스카우팅과 별도). 리그 단위로 영웅 픽·밴·승률을 본다. 각 뷰에 **역할 필터(전체/탱커/딜러/서포터)** 와 **Top N(20/50/전체)** 필터를 공통 제공한다.

| 뷰 | 명세 | 상태 |
|----|------|------|
| 개요 | 17.1 | 카운터 + 밴 상위(지금) + 픽 상위·포지션 픽률(데이터 따라) |
| **밴 메타** | 17.7 | 글로벌 밴, 선밴/후밴 분리, 팀별 밴, 모드별 밴 — **핵심, 전부 지금 동작** |
| 맵 메타 | 17.2 | 맵별 밴·팀 승률(지금) + 맵별 픽(데이터 따라) |
| 팀 분석 | 17.3 | 팀별 밴 성향(지금) + 영웅 픽·승률(데이터 따라) |
| 선수 분석 | 17.4 | 선수별 주영웅·승률, 표본 10 미만 ⚠ |
| 승률 분석 | 17.5 | 글로벌 영웅 승률, 승자 미상 세트 제외 |
| 포지션 픽률 | 17.6 | 역할 슬롯(DPS=0·1, Tank=2, Support=3·4) 기준 픽 점유율 |
| 데이터 노트 | 17.8 | 지표 정의(18)·전제(16) 노출 |

### 지표 정의 (§18)

- **pick_count** (18.1): (맵세트, 팀) 단위 중복제거 영웅 등장 수. 같은 세트·팀의 동일 영웅은 1.
- **pick_rate** (18.2): 같은 역할/포지션 범위 내 pick_count 비율(%).
- **ban_count** (18.3): (맵세트) 단위 밴 등장 수. 선밴/후밴 **합산·분리** 둘 다 제공.
- **matches_played** (18.4): 영웅이 픽된 맵세트 수. **win_rate** = wins / matches_played, 무승부는 played 포함·win 제외 (18.5).
- **저표본** (18.6): 맵세트 10 미만 → ⚠ 경고. 승자 미상·승점 미기록 세트는 승률 집계 제외 (18.7).
- 모든 픽 지표는 **"선픽(오프닝) 기준"** 으로 라벨링한다 — 참고 사이트의 "세트 내 전체 등장 픽"과 다르다 (16.4·21.3).

### 지금 구현 vs 데이터 대기 (§20)

- **지금 바로 동작**: 밴 메타 전부(17.7), 맵별 팀 승률·맵별 밴(17.2 일부), 개요의 밴 카운터·밴 상위(17.1 일부), 팀별 밴 성향(17.3 일부).
- **데이터 대기**(빈 표 대신 "선픽 데이터 대기 — 채울 컬럼: 상수팀 첫픽 / 하수팀 첫픽" 안내): 픽 카운터·픽 상위·포지션 픽률, 맵별 픽, 팀별 픽·영웅 승률, 선수 분석, 승률 분석.
- 단, **현재 시트엔 선픽이 상당수 입력되어 있어** 픽 화면 다수가 이미 채워져 표시된다.

## 데이터 파싱 규칙 (명세 8장)

- 푸시 점수는 거리 문자열(`144.86m`)과 정수 점수가 섞임 → `m` 이면 거리, 거리는 큰 값이 승. `144` 이상을 풀푸시로 추정.
- `mapPicker`가 `ADMIN`/공란인 행은 "맵 선호" 집계에서 제외(다른 집계엔 포함).
- 영웅 악센트(`Lúcio`, `Torbjörn`)·팀명 공백·대소문자는 NFC 정규화.
- 매치 점수는 세트 합산 vs 대진표를 비교해 불일치 시 **서버 콘솔 경고**(`[교차검증]`). 순위표 매치 W/L 도 파생값과 교차검증.

## 갱신·캐시

- ISR로 **10분(600초)** 마다 시트를 재검증한다. 헤더의 새로고침 버튼은 클라이언트 재요청이며, 캐시 주기 안에서는 직전 캐시를 보여줄 수 있다.
- 즉시 강제 갱신이 필요하면 Vercel 재배포 또는 On-Demand Revalidation 도입을 고려한다(향후 확장).

## 향후 (명세 12.5 / 10.4)

- 세트 단위 로스터가 충분히 쌓이면 승률 추정을 로지스틱 회귀로 교체할 수 있도록 입력 피처/출력 인터페이스를 분리해 두었다.
- 공개가 필요하면 인증 게이트 추가(현재 `noindex`만 적용).
